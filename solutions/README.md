# Solutions

Reference answers to weekly exercises and projects.

## The peek policy

This folder is **last resort**, not first stop. Read this paragraph carefully because it's the difference between learning Python and pretending to learn Python:

1. **Try the exercise yourself.** Even if it feels impossible.
2. **Get stuck for at least 15 minutes.** That uncomfortable feeling is your brain wiring up new pathways. Don't short-circuit it.
3. **Re-read the lesson.** Then search Google or the official docs.
4. **Ask an AI to explain the error**, not to write the code. Phrasing matters:
   > "Here's my code and the error. What is Python actually telling me? Don't write a fix — just explain."
5. **Only then**, peek at the solution. Compare line by line. If your version is different but works, great — there are many right answers.

## Why the gating

Reading a solution feels like progress. It isn't. You're just watching someone else type. Coding fluency comes from the **struggle of producing code yourself**.
